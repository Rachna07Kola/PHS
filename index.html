1. Hiring Assistant AI ‚Äì Agentic Resume Screening System (Academic Project)

This intelligent hiring assistant was built to transform traditional recruitment by integrating agentic AI with Retrieval-Augmented Generation (RAG). It autonomously screens and ranks resumes, engages recruiters through dynamic chat, and evolves based on continuous feedback‚Äîall in a privacy-first, offline-ready architecture.

Objective: To develop a lightweight, privacy-compliant, fully offline RAG-powered AI assistant that reduces hiring friction, enhances fairness, and adapts to evolving recruiter criteria in real-time.

Core Features:

  • Resume Parsing: PyMuPDF-based extraction from PDFs (resumes & job descriptions)
  • Semantic Matching: SentenceTransformers embeddings + in-memory similarity for context-rich matching
  • Hybrid Scoring: Combines semantic similarity with rubric-based skill match using Pandas
  • Recruiter Interaction: Dynamic conversations powered by quantized Mistral-7B-Instruct model
  • Feedback Loop: Updates scoring logic and rubric based on recruiter input

Technologies:

  • Python 3.x for backend logic and processing
  • Streamlit: Interface for recruiter interaction
  • PyMuPDF: For text extraction
  • SentenceTransformers: all-MiniLM-L6-v2 embeddings
  • Pandas: Skill rubric matching & hybrid ranking
  • ctransformers: For local inference using quantized Mistral-7B-Instruct
  • RAG: Integrated retrieval + reasoning for enhanced explainability
  • Offline Mode: Fully local, secure, and cloud-free operation

Key Outcomes:

  • Efficiency: Reduced time-to-hire by automating resume triage and ranking
  • Transparency: Provided clear explanations behind matches and rankings
  • Adaptability: Evolved in real-time based on recruiter input and feedback
  • Privacy-Focused: Local inference ensured no external dependencies

3. Financial Fraud Detection ‚Äî Machine Learning Model (Academic Project)

This project addressed the escalating challenge of financial fraud using supervised machine learning. By applying multiple models on a labeled dataset, the system aimed to detect suspicious transactions effectively, prioritizing recall to minimize undetected fraud and integrating fairness metrics to reduce bias.

Approach & Methodology:

  • Matplotlib & Seaborn for visualization

  • Reduced false negatives by 15% compared to baseline models
